# Cronograma del proyecto

El cronograma del Sistema de Gestión de Eventos Universitarios se organizó utilizando el principio W5HH, con el propósito de definir claramente qué se realizará, por qué, cuándo, quiénes participarán, dónde se desarrollará, cómo se ejecutará y cuánto esfuerzo requerirá.

## Principio W5HH

| Elemento | Descripción |
|---|---|
| What (¿Qué?) | Desarrollar un sistema que facilite la gestión de eventos universitarios, el registro de usuarios y la inscripción de estudiantes. |
| Why (¿Por qué?) | Optimizar la organización de eventos y facilitar el acceso de los estudiantes a la información. |
| When (¿Cuándo?) | Durante el semestre académico, siguiendo el cronograma establecido por la asignatura. |
| Who (¿Quién?) | Los integrantes del equipo de desarrollo, responsables del análisis, diseño, programación, pruebas y documentación. |
| Where (¿Dónde?) | El desarrollo se realiza mediante herramientas colaborativas y un repositorio público de GitHub. |
| How (¿Cómo?) | Utilizando metodología ágil Scrum, tecnologías web y control de versiones con GitHub. |
| How Much (¿Cuánto costará?) | El proyecto se desarrolla como trabajo académico, considerando principalmente el tiempo y esfuerzo del equipo. |

## Etapas del proyecto

| Etapa | Actividad |
|---|---|
| Análisis | Identificación del problema y definición de requerimientos. |
| Diseño | Elaboración de la estructura e interfaz del sistema. |
| Desarrollo | Implementación de las funcionalidades con HTML, CSS y JavaScript. |
| Pruebas | Verificación del registro, gestión e inscripción a eventos. |
| Documentación | Elaboración del informe, README y evidencias del prototipo. |
| Entrega | Publicación del repositorio y presentación del sistema funcional. |
