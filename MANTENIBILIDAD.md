# Mantenibilidad del sistema

La mantenibilidad permite corregir errores, adaptar el sistema a nuevos cambios y agregar funcionalidades después de su implementación.

## Mantenimiento correctivo

Permite corregir errores detectados durante el uso del sistema.

Algunos ejemplos son:

- Errores en el registro de usuarios.
- Problemas en el inicio de sesión.
- Fallos durante la creación o inscripción a eventos.

## Mantenimiento adaptativo

Permite realizar modificaciones cuando cambien los procesos, tecnologías o necesidades de la universidad.

Algunos ejemplos son:

- Adaptar el sistema a nuevas políticas institucionales.
- Actualizar el sistema para nuevos navegadores.
- Modificar los datos solicitados para los eventos.

## Mantenimiento perfectivo

Permite mejorar el funcionamiento del sistema y agregar nuevas características.

Algunos ejemplos son:

- Agregar notificaciones de eventos.
- Mejorar el rendimiento de la aplicación.
- Incorporar nuevas opciones para estudiantes y organizadores.

## Estimación básica

| Etapa | Tiempo estimado | Costo estimado |
|---|---:|---:|
| Corrección de errores | 1 semana | $100 |
| Actualización del sistema | 2 semanas | $150 |
| Nuevas funcionalidades | 4 semanas | $300 |
