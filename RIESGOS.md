# Matriz de riesgos del proyecto

La gestión de riesgos permite identificar posibles problemas que podrían afectar el desarrollo del Sistema de Gestión de Eventos Universitarios y establecer acciones para prevenirlos o reducir su impacto.

| ID | Riesgo | Probabilidad | Impacto | Plan de respuesta |
|---|---|---|---|---|
| R1 | Retraso en el desarrollo del sistema | Media | Alto | Establecer un cronograma semanal y dar seguimiento al avance de las actividades. |
| R2 | Pérdida de información del proyecto | Baja | Alto | Realizar copias de seguridad periódicas y utilizar GitHub para conservar las versiones del proyecto. |
| R3 | Errores durante las pruebas | Media | Medio | Ejecutar casos de prueba para cada funcionalidad y corregir los errores detectados. |
| R4 | Falta de comunicación entre los integrantes | Baja | Medio | Programar reuniones periódicas para revisar tareas, avances y dificultades. |
| R5 | Cambios en los requerimientos del sistema | Media | Alto | Revisar y validar los requerimientos antes de realizar cambios en el desarrollo. |

## Seguimiento

Los riesgos identificados deberán revisarse durante el desarrollo del proyecto. En caso de que alguno ocurra, se aplicará el plan de respuesta establecido para reducir sus consecuencias.
