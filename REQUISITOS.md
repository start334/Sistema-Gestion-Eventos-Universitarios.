# Requisitos del sistema

## Requisitos funcionales

### RF-01 Registro de usuarios
El sistema permite registrar estudiantes y organizadores mediante un formulario con sus datos personales.

### RF-02 Gestión de eventos
El organizador puede crear, editar, consultar y eliminar eventos universitarios.

### RF-03 Inscripción a eventos
Los estudiantes pueden consultar los eventos disponibles e inscribirse cuando existan cupos.

## Requisitos no funcionales

### RNF-01 Seguridad
El sistema permite el acceso únicamente a usuarios autenticados mediante correo y contraseña.

### RNF-02 Rendimiento
El sistema responde a las solicitudes del usuario en un tiempo máximo de tres segundos.

### RNF-03 Disponibilidad
El sistema está disponible para el registro, consulta e inscripción a eventos.
